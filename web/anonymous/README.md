# Anonymous Upload System

Secure, token-based file upload system that allows controlled anonymous file uploads without exposing public endpoints.

## 🔐 Security Model

Instead of allowing anyone to upload files, this system uses **one-time use tokens** that:

- Must be generated by authenticated administrators
- Can be configured with specific limits (uploads, expiry, target directory)
- Are automatically revoked after use or expiration
- Prevent unauthorized access

## 🚀 Usage Flow

### For Administrators

1. **Access Token Manager**
   - Navigate to: `http://localhost:8080/admin/tokens`
   - Requires authentication

2. **Generate Upload Token**
   - Set target directory (where files will be uploaded)
   - Set maximum uploads (default: 1 for one-time use)
   - Set expiry hours (default: 24 hours)
   - Click "Generate Token"

3. **Share Upload Link**
   - Copy the generated upload URL
   - Share with users who need to upload files
   - Token is embedded in the URL

### For Users (Anonymous)

1. **Access Upload Page**
   - Click the shared upload URL (contains token)
   - OR navigate to `/upload` and enter token manually

2. **Upload Files**
   - Token is automatically validated
   - Drag and drop files or click to browse
   - Monitor real-time upload progress
   - Files are uploaded to admin-specified directory

3. **Token Expiration**
   - Token is automatically revoked after successful upload(s)
   - Or expires after configured time period

## 🔌 API Endpoints

### Token-Based Upload (No Auth, Token Required)

#### Upload File (Direct)

```
POST /upload/token/file?token={token}
Content-Type: multipart/form-data

Response: 201 Created
{
  "message": "Upload successful",
  "uploads_remaining": 0
}
```

#### Chunked Upload

**1. Initialize:**

```
POST /upload/token/chunk/start?token={token}&filename={filename}
Response: 200 OK
```

**2. Upload Chunks:**

```
POST /upload/token/chunk?token={token}&filename={filename}&chunk_index={index}
Content-Type: application/octet-stream
Body: chunk data

Response: 200 OK
```

**3. Complete:**

```
POST /upload/token/chunk/complete?token={token}&filename={filename}
Response: 200 OK
```

#### Validate Token

```
GET /upload/token/validate?token={token}

Response: 200 OK
{
  "valid": true,
  "target_dir": "/uploads",
  "max_uploads": 1,
  "uploads_remaining": 1,
  "expires_at": "2026-02-12T10:30:00Z"
}
```

### Admin Endpoints (Auth Required)

#### Generate Token

```
POST /api/upload-tokens/generate?target_dir=/uploads&max_uploads=1&expiry_hours=24

Response: 200 OK
{
  "token": "abc123...",
  "upload_url": "localhost:8080/upload?token=abc123...",
  "target_dir": "/uploads",
  "max_uploads": 1,
  "expiry_hours": 24,
  "max_file_size": 11811160064
}
```

#### List Active Tokens

```
GET /api/upload-tokens/list

  "tokens": [...],
  "count": 5
}
```

#### Get Token Info

```
GET /api/upload-tokens/info?token={token}
```

## ⚙️ Configuration

### Token Parameters

| Parameter       | Default    | Description                            |
| --------------- | ---------- | -------------------------------------- |
| `target_dir`    | `/uploads` | Directory where files will be uploaded |
| `max_uploads`   | `1`        | Number of files that can be uploaded   |
| `expiry_hours`  | `24`       | Token validity period in hours         |
| `max_file_size` | `11GB`     | Maximum file size per upload           |

### Token Behavior

- **One-Time Use**: Tokens with `max_uploads=1` are revoked after first upload
- **Auto-Expiration**: Tokens expire after configured hours
- **Auto-Cleanup**: Expired tokens are automatically removed every hour
- **Single Upload Tracking**: Each successful upload increments the counter

## 🔒 Security Features

### Token-Based Access

✅ Generates cryptographically secure random tokens (64 hex characters)  
✅ Tokens must be validated before each upload operation  
✅ Invalid/expired/revoked tokens return 403 Forbidden

### Upload Limits

✅ Maximum upload count per token  
✅ File size limits (default 11GB)  
✅ Time-based expiration

### Path Security

✅ Path traversal protection (`ExpelDotPath`)  
✅ Admin-controlled target directories  
✅ No user-specified paths

### Automatic Cleanup

✅ Tokens auto-revoke after max uploads reached  
✅ Expired tokens cleaned up hourly  
✅ Manual revocation available

```go
    // ... other routes ...

    // Register anonymous upload routes
```

### Generate Token Programmatically

```go
import "aaxion/internal/anonymous_upload"

token, err := anonymous_upload.GenerateUploadToken(
    "/uploads/project-x",  // target directory
    5,                      // max uploads
    48,                     // expiry hours
    5 << 30,               // max file size (5GB)
)
```

### Validate Token

```go
tokenInfo, err := anonymous_upload.ValidateToken(token)
if err != nil {
    // Token invalid, expired, or revoked
    return
}
// Use tokenInfo.TargetDir, etc.
```

## 🐛 Troubleshooting

### "Invalid or expired token"

- Token may have been used (one-time tokens)
- Token may have expired
- Token may have been revoked by admin
- Solution: Generate new token

### "Maximum uploads reached"

- Token's upload limit has been reached
- Solution: Generate new token or increase `max_uploads`

### "Missing upload token"

- Token not provided in request
- Solution: Ensure URL includes `?token=...` parameter

### Upload fails immediately

- Check target directory exists
- Verify directory permissions
- Check disk space
- Review server logs

## 📝 Migration from Public Upload

If you were using the old public upload system:

**Old (Insecure):**

```
POST /files/upload/public?dir=/uploads
→ Anyone can upload to any directory
```

**New (Secure):**

```
1. Admin generates token with target_dir=/uploads
2. POST /upload/token/file?token={token}
→ Only users with valid token can upload
→ Files go to admin-specified directory only
```

## 🤝 Best Practices

1. **Use Short Expiry Times**: Set `expiry_hours` to minimum needed
2. **One-Time Tokens**: Use `max_uploads=1` for sensitive uploads
3. **Specific Directories**: Don't use root or system directories
4. **Monitor Active Tokens**: Regularly review and revoke unused tokens
5. **Share Securely**: Send upload links through secure channels
6. **Immediate Revocation**: Revoke tokens immediately after use if time-sensitive
